# ASR-PRO 导航语音固件烧录说明

## 语音命令定义

| ID | 语音命令 | 串口输出 | 行为 |
|----|---------|---------|------|
| 0 | 智能管家（唤醒词） | `WAKE` | 唤醒设备，回复"我在，请说指令" |
| 1 | 去一号 | `NAV:1` | 导航到一号点 |
| 2 | 去二号 | `NAV:2` | 导航到二号点 |
| 3 | 去三号 | `NAV:3` | 导航到三号点 |
| 4 | 返回 | `NAV:0` | 返回起始点 |
| 5 | 回家 | `NAV:0` | 返回起始点（同ID 4） |
| 6 | 停止 | `STOP` | 停止/停车 |
| 7 | 停车 | `STOP` | 停车（同ID 6） |
| 8 | 自动模式 | `MODE:ROS` | 切换到ROS2控制模式 |
| 9 | 手动模式 | `MODE:MANUAL` | 切换手动模式 |

## 烧录步骤

### 1. 安装天问Block（Windows）
   - 下载地址：天问Block官网 http://www.haohaodada.com/
   - 或使用项目中的安装包：`软件/天问Block V2024 夏季正式版 SP01.exe`
   - 安装后注册账号并绑定手机号（生成语音模型需要云端服务）

### 2. 打开项目
   - 启动天问Block
   - 选择设备型号：ASRPRO
   - 打开项目文件：`firmware/asrpro_nav.hd`

### 3. 配置串口
   - 确认 ASR_CODE 中的 Serial 输出使用 **UART1**（PA2-TX, PA3-RX）
   - 波特率：9600
   - 如需修改，在天问Block的"设置"中调整串口参数

### 4. 编译并下载
   - 连接 ASR-PRO 开发板到电脑（USB转TTL或STC-LINK）
   - 点击"编译"按钮（自动生成语音模型并编译固件）
   - 点击"下载"按钮烧录到芯片

### 5. 接线到Linux主机
   - ASR-PRO UART1 TX (PA2) → USB-TTL模块 RX
   - ASR-PRO UART1 RX (PA3) → USB-TTL模块 TX
   - GND → GND
   - USB-TTL 插入 Linux 主机
   - 确认设备路径（通常是 `/dev/ttyUSB2`）

### 6. 测试
   - 启动 asr_bridge 节点：
     ```
     ros2 launch asr_bridge asr_bridge.launch.py asr_port:=/dev/ttyUSB2
     ```
   - 对 ASR-PRO 说"智能管家"唤醒
   - 说"去一号"看ROS2日志输出

## 串口协议

- 物理层：UART, 9600bps, 8N1
- 帧格式：`<命令>[\n]`
- 命令类型：
  - `NAV:<point_id>` - 导航到任务点
  - `STOP` - 停止
  - `MODE:<mode>` - 切换模式
  - `WAKE` - 唤醒
- 应答（上位机→ASR-PRO）：
  - `OK:<cmd>` - 命令已执行
  - `ERR:NOPOINT:<id>` - 任务点不存在
- 启动握手：
  - 上位机启动时会发送 `BOOT:OK` 确认连接

## 扩展命令词

如需增加更多命令词，在天问Block中：
1. 在 setup 块中添加 `asr_id` 块，指定新的语音文字和ID
2. 在 ASR_CODE 的 switch/case 中添加对应ID的 case，输出自定义串口字符串
3. 在 `config/commands.yaml` 中注册新的命令映射
4. 重新编译下载固件

## 注意事项

- ASR-PRO 为 3.3V 逻辑电平，与树莓派可以直接连接
- 如果连接5V系统（如Arduino），需在TX引脚加限流电阻或使用电平转换
- 命令词总量上限：200条（2M Flash）/ 300条（4M Flash）
- 如果烧录时报错，检查：驱动安装（CH340/STC-LINK）、USB线（需数据线非纯充电线）
- 烧录报错另见：`ASR-PRO核心板下载报错解决方案.jpg`
